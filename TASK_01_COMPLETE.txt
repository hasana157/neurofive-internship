FEW-SHOT PROMPT:
================================================================================

Classify the following customer support messages into one of three categories: 
"Complaint", "Question", or "Praise".

Examples:
- "Your product is amazing!" → Praise
- "How do I change my settings?" → Question
- "This is broken and I'm furious!" → Complaint

Now classify these messages:

Message 1: Your product broke after 2 days! This is unacceptable!
Message 2: How do I reset my password?
Message 3: Amazing customer service, thank you so much!

Message 10: Thanks for fixing that bug so quickly!

For each message, provide only the classification.

PROMPT DESIGN NOTES:

Zero-Shot Design:
- No examples provided
- Direct instruction only
- Tests model's inherent understanding
- Simpler, more concise

Few-Shot Design:
- 3 diverse examples provided
- Covers all 3 categories
- Clear input-output format with arrow notation
- Guides model behavior without complex instructions

Both prompts:
- Same 10 test messages
- Same output format requirement (single classification)
- Same three categories (Complaint, Question, Praise)
- Identical test dataset for fair comparison

================================================================================
 DETAILED ANALYSIS & INTERPRETATION
================================================================================

EXPERIMENT OVERVIEW:
- Models Tested: Claude, ChatGPT, Gemini
- Task Type: Multi-class Classification (3 categories)
- Sample Size: 10 messages
- Approach 1: Zero-Shot (no examples)
- Approach 2: Few-Shot (3 examples)

WHY BOTH APPROACHES ACHIEVED 100% ACCURACY:

1. TASK SIMPLICITY
   - The classification task was straightforward
   - Clear semantic distinctions between categories
   - No edge cases or ambiguous messages
   - Messages had strong linguistic signals

2. MODEL CAPABILITY
   - Modern LLMs (Claude, GPT-4, Gemini) are highly sophisticated
   - Can infer classification rules from category descriptions alone
   - Extensive training on similar tasks
   - Strong zero-shot capabilities on basic NLP tasks

3. MESSAGE CLARITY
   - Test messages were explicit and unambiguous
   - Each message strongly favored one category
   - No sarcasm, irony, or mixed sentiments
   - Clear linguistic indicators present

WHY FEW-SHOT REMAINS ESSENTIAL:

Despite perfect accuracy in both cases, few-shot prompting is superior for:

1. PRODUCTION RELIABILITY
   - Reduces hallucination in edge cases
   - Ensures consistent output format
   - Provides explicit guidance to model
   - Increases reproducibility

2. COMPLEX SCENARIOS
   - Helps with ambiguous or nuanced messages
   - Critical for sensitive classifications
   - Reduces model drift over time
   - Better for multi-step reasoning

3. COST EFFICIENCY
   - Reduces errors that require correction
   - Minimizes need for manual review
   - Prevents expensive mistakes
   - Improves workflow efficiency

4. USER CONTROL
   - Gives humans more influence over outputs
   - Makes model behavior more predictable
   - Easier to debug when issues arise
   - Better documentation of intent
